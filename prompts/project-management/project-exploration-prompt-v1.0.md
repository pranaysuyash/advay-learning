# Project Exploration & Innovation Prompt (Generic)

**Version:** v1.0
**Purpose:**
Systematically surface, track, and explore unresolved questions, unexplored features, and strategic opportunities for any software project. This prompt provides a comprehensive framework for innovation, compliance, and strategic planning.

---

## Core Principles

1. **Evidence-Based**: All suggestions backed by project context, user feedback, or industry best practices
2. **Systematic**: Follow structured exploration methodology
3. **Comprehensive**: Cover all major project dimensions (technical, business, user experience, compliance)
4. **Actionable**: Result in concrete next steps, tickets, or research plans
5. **Neutral**: Applicable to any software project regardless of domain or technology stack

---

## Usage Instructions

### When to Use

- Quarterly planning and roadmap reviews
- Innovation sprints or hackathons
- Before major releases or pivots
- When stakeholders ask "what's next?" or "what are we missing?"
- During project reassessments or bottlenecks

### How to Use

1. **Preparation**: Gather project context (docs, tickets, user feedback, analytics)
2. **Exploration**: Work through each category systematically
3. **Documentation**: Update project backlog with findings
4. **Prioritization**: Rank opportunities by impact vs effort
5. **Action**: Create tickets, research plans, or prototypes

### Output Format

- **Unresolved Questions**: List blocking, medium, and low-priority questions
- **Unexplored Features**: Technical, business, and user experience opportunities
- **Strategic Opportunities**: Compliance, scaling, and business development
- **Next Actions**: Concrete tickets, research, or implementation plans

## Exploration Framework

### 1. Unresolved Questions & Blockers

**Goal**: Identify and prioritize questions preventing progress

**Checklist**:

- Review open questions in project documentation
- Check worklog tickets for blocked items
- Scan for technical debt or architectural decisions pending
- Identify stakeholder disagreements or unclear requirements

**Categories**:

- **Technical**: Architecture decisions, technology choices, performance bottlenecks
- **Business**: Monetization, market positioning, competitive analysis
- **Compliance**: Legal requirements, privacy regulations, accessibility standards
- **User Experience**: Usability issues, feature requests, user feedback gaps

### 2. Feature & Product Opportunities

**Goal**: Surface new features, improvements, and product directions

**Checklist**:

- Analyze user feedback and support tickets
- Review competitor features and market trends
- Brainstorm based on core value proposition
- Consider adjacent use cases and market expansion

**Categories**:

- **Core Features**: Enhancements to existing functionality
- **New Modules**: Additional capabilities or product lines
- **Integration**: Third-party services, APIs, or platforms
- **Gamification**: User engagement and retention features
- **Personalization**: Adaptive, AI-driven, or customized experiences

### 3. Technical Hardening & Infrastructure

**Goal**: Identify technical improvements and infrastructure needs

**Checklist**:

- Review performance metrics and bottlenecks
- Assess security posture and vulnerabilities
- Evaluate scalability and reliability
- Check monitoring, logging, and observability

**Categories**:

- **Performance**: Optimization, caching, async processing
- **Security**: Authentication, authorization, data protection
- **Scalability**: Load balancing, database optimization, cloud migration
- **Monitoring**: Logging, alerting, error tracking, analytics
- **DevOps**: CI/CD, testing automation, deployment processes

### 4. Compliance & Legal

**Goal**: Ensure regulatory compliance and risk mitigation

**Checklist**:

- Review applicable regulations (GDPR, HIPAA, COPPA, etc.)
- Assess data handling and privacy practices
- Check accessibility and inclusivity standards
- Evaluate business continuity and disaster recovery

**Categories**:

- **Privacy**: Data collection, consent, user rights
- **Security**: Encryption, access controls, audit logging
- **Accessibility**: WCAG compliance, inclusive design
- **Legal**: Terms of service, intellectual property, contracts

### 5. User Experience & Design

**Goal**: Improve user satisfaction and engagement

**Checklist**:

- Analyze user behavior data and feedback
- Review usability testing results
- Assess onboarding and retention metrics
- Consider accessibility and internationalization

**Categories**:

- **Onboarding**: First-time user experience, tutorials
- **Navigation**: Information architecture, user flows
- **Feedback**: Error handling, notifications, help systems
- **Accessibility**: Multi-language, screen readers, keyboard navigation
- **Mobile/Responsive**: Cross-device compatibility

### 6. Business & Growth Opportunities

**Goal**: Identify revenue, partnership, and expansion opportunities

**Checklist**:

- Analyze market trends and competitive landscape
- Review user acquisition and retention metrics
- Assess partnership and integration opportunities
- Consider pricing and monetization strategies

**Categories**:

- **Monetization**: Subscription models, freemium, premium features
- **Partnerships**: Integrations, white-labeling, co-marketing
- **Market Expansion**: New user segments, geographies, use cases
- **Brand**: Marketing, community building, thought leadership

### 7. Process & Team Improvements

**Goal**: Enhance development processes and team productivity

**Checklist**:

- Review development workflows and bottlenecks
- Assess team communication and collaboration
- Evaluate quality assurance and testing processes
- Consider knowledge sharing and documentation

**Categories**:

- **Development**: Code quality, testing, deployment automation
- **Collaboration**: Communication tools, project management
- **Documentation**: API docs, user guides, internal knowledge base
- **Quality**: Code reviews, testing strategies, QA processes

### 8. Risk Assessment & Mitigation

**Goal**: Identify and plan for potential risks

**Checklist**:

- Review technical debt and maintenance burden
- Assess vendor dependencies and lock-in risks
- Evaluate team knowledge and bus factor
- Consider market and competitive risks

**Categories**:

- **Technical Risks**: Outdated dependencies, architectural issues
- **Business Risks**: Market changes, competitive threats
- **Operational Risks**: Team changes, vendor issues
- **Compliance Risks**: Regulatory changes, legal issues

## Prioritization Framework

### Impact vs Effort Matrix

- **High Impact, Low Effort**: Quick wins, implement immediately
- **High Impact, High Effort**: Strategic initiatives, plan carefully
- **Low Impact, Low Effort**: Nice-to-haves, implement when resources available
- **Low Impact, High Effort**: Avoid or deprioritize

### Risk-Adjusted Prioritization

- **High Risk, High Reward**: Research thoroughly before committing
- **Low Risk, High Reward**: Safe bets, good candidates for experimentation
- **High Risk, Low Reward**: Avoid unless strategically necessary
- **Low Risk, Low Reward**: Consider for team morale or learning opportunities

## Documentation & Tracking

### Required Outputs

1. **Exploration Report**: Summary of findings with prioritized recommendations
2. **Backlog Updates**: Add new items to project exploration backlog
3. **Ticket Creation**: Convert high-priority items into actionable tickets
4. **Research Plans**: Define next steps for uncertain or high-risk items

### Living Documentation

- **Project Exploration Backlog**: Central repository for all opportunities
- **Worklog Tickets**: Track implementation of exploration findings
- **Research Documents**: Detailed analysis of complex opportunities
- **Roadmap Updates**: Incorporate findings into strategic planning

## Example Implementation

### For a SaaS Product

1. **Unresolved Questions**: API versioning strategy, data retention policies
2. **Feature Opportunities**: Mobile app, advanced analytics dashboard
3. **Technical Hardening**: Database optimization, CDN implementation
4. **Compliance**: SOC 2 certification, GDPR compliance review
5. **User Experience**: Onboarding flow improvements, help system
6. **Business Growth**: Enterprise tier, partnership program
7. **Process**: CI/CD improvements, automated testing expansion
8. **Risks**: Vendor dependency assessment, team scaling plan

### For a Mobile App

1. **Unresolved Questions**: Push notification strategy, offline mode requirements
2. **Feature Opportunities**: Social features, premium subscriptions
3. **Technical Hardening**: Crash reporting, performance monitoring
4. **Compliance**: App store guidelines, privacy policy updates
5. **User Experience**: Dark mode, gesture navigation
6. **Business Growth**: In-app purchases, affiliate marketing
7. **Process**: Code signing automation, beta testing workflow
8. **Risks**: Platform dependency, user acquisition costs

- **Quality**: Percentage of opportunities that become implemented features
- **Impact**: Business value generated from implemented opportunities
- **Efficiency**: Time from exploration to implementation decision
