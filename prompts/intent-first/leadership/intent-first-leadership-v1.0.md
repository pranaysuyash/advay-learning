# Intent-First Leadership v1.0

## "Lead with Clarity, Not Control"

**Core Principle:** Align people around clear intent, constraints, and context so they can move fast with good judgment.

**Codebase-First Focus:** Build upon existing team strengths and processes. Enhance current capabilities rather than imposing wholesale changes.

---

## Universal Investigation Framework

### Phase 1: Context Discovery

1. **Define the situation and stakeholders**
2. **Gather signals, history, and constraints**
3. **Surface hidden incentives and risks**
4. **Review existing team capabilities** (what's already working well?)

### Phase 2: Intent Analysis

- What outcome are we truly optimizing for?
- Which principles must not be violated?
- What trade-offs are acceptable?
- **What existing strengths can we build upon?**

### Phase 3: Priority Assessment

- **Impact**: magnitude of upside or avoided downside
- **Effort**: time, energy, attention, capital
- **Risk**: reversibility, blast radius, optionality
- **Team Fit**: alignment with existing capabilities

---

## Decision Levers

- **Clarity:** narrative, goals, non-goals
- **Cadence:** rituals, review loops, escalation paths
- **Constraints:** budget, headcount, compliance, timebox
- **Culture:** defaults for ownership, feedback, learning
- **Capabilities:** existing team strengths to leverage

---

## Quick Filter

Skip deep work if all are true:

- Low, vague, or misaligned intent
- High effort for marginal value
- Better alternatives exist now
- **Team already has capability to handle this**

→ **Log as intent debt** (record context, revisit trigger).

---

## Codebase-First Enhancement Rule

Ship the smallest action that:

1. **Builds on existing strengths** - leverage what the team does well
2. **Proves intent** - demonstrates clear direction
3. **Creates real, observable value** - tangible outcomes
4. **Enables team autonomy** - context for good judgment
5. **Preserves working processes** - don't break what's working

Instrument for feedback, then iterate deliberately.

---

## Priority Matrix

| Impact | Effort | Risk | Team Fit | Priority |
|---|---|---|---|---|
| High | Low–Med | Low | Strong | **Do now, empower owners** |
| High | High | Med | Strong | **Timebox, split, delegate** |
| High | High | Med | Weak | **Build capability first, then execute** |
| Med | Low | Low | Strong | **Batch into weekly review** |
| Low | Any | Any | Any | **Log as leadership debt** |

## Ready-to-Use AI Prompt

```
You are my Leadership Intent Agent. Apply the Intent-First Leadership Philosophy.

### Context
[describe situation]

### Investigation
1) Clarify true outcome and constraints
2) Identify existing team capabilities and strengths
3) Identify 2–3 highest-leverage moves that build on existing foundation
4) Map risks, reversibility, and optionality
5) Propose an incremental action with metrics

### Output
- Intent summary
- Existing capabilities to leverage
- Options compared (impact, effort, risk, team fit)
- Recommended incremental action
- 2 follow-on iterations
- Metrics & review cadence
```

## Key Questions

- **"What would a good-enough version look like today?"**
- **"What existing team capabilities can we build upon?"**
- **"What if I did nothing for two weeks?"**
- **"How can we learn the same thing faster and cheaper?"**
- **"What would I regret not trying?"**
- **"How do I provide context without controlling execution?"**
