## Problem

**92% of games (36/39) lack proper error handling.**

When errors occur:
- Games show blank screens
- No retry option for users
- No error logging for debugging
- Kids/parents have no recourse

---

## Scope Contract

**In-scope**:
- Add try/catch blocks to critical operations
- Add error boundaries around games
- Add user-friendly error messages
- Add retry functionality

**Out-of-scope**:
- Backend error handling (separate initiative)

**Behavior change allowed**: YES (errors will be handled gracefully)

---

## Technical Design

### 1. Wrap Games in Error Boundary

```typescript
// Already exists: src/frontend/src/components/errors/GlobalErrorBoundary.tsx
// Use in App.tsx routes:
<Route
  path='/games/my-game'
  element={
    <ProtectedRoute>
      <GlobalErrorBoundary>
        <CameraSafeRoute gameName='My Game'>
          <MyGame />
        </CameraSafeRoute>
      </GlobalErrorBoundary>
    </ProtectedRoute>
  }
/>
```

### 2. Add Try/Catch in Game Logic

```typescript
try {
  await someAsyncOperation();
} catch (error) {
  console.error('Game error:', error);
  setErrorState({
    message: 'Something went wrong. Try again?',
    canRetry: true,
  });
}
```

### 3. Add Error UI

```typescript
if (error) {
  return (
    <GameContainer>
      <ErrorState 
        error={error} 
        onRetry={() => setErrorState(null)} 
      />
    </GameContainer>
  );
}
```
