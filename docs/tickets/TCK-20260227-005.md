# TCK-20260227-005 :: Add Reduce Motion Support to All Games

**Type**: ACCESSIBILITY_FIX  
**Owner**: Unassigned  
**Created**: 2026-02-27  
**Status**: OPEN

**Severity**: Accessibility violation - WCAG 2.1 AA

---

## Problem

**87% of games (34/39) don't support reduce motion preference.**

Kids with vestibular disorders, motion sensitivity, or migraines experience:
- Dizziness
- Nausea
- Headaches
- Cannot use the app

**Legal Risk**: WCAG 2.1 AA violation

---

## Scope Contract

**In-scope**:
- Add `useReducedMotion` hook to all games
- Disable/reduce animations when preferred
- Maintain functionality without motion

**Out-of-scope**:
- Removing all animations (just reduce them)
- Audio/tactile feedback changes

**Behavior change allowed**: YES (animations will be reduced for some users)

---

## Technical Design

### Pattern to Add

```typescript
import { useReducedMotion } from 'framer-motion';

const reducedMotion = useReducedMotion();

// In animation props:
<motion.div
  animate={{ opacity: 1, scale: 1 }}
  initial={{ opacity: 0, scale: 0.8 }}
  transition={reducedMotion 
    ? { duration: 0.01 }  // Instant
    : { duration: 0.3, ease: 'easeOut' }
  }
/>
```

### For Confetti/Particles

```typescript
import confetti from 'canvas-confetti';

if (!reducedMotion) {
  confetti({
    particleCount: 100,
    spread: 70,
  });
} else {
  // Show static success message instead
  setShowSuccessMessage(true);
}
```
